# 字典系统与 JSON 配置职责边界说明

## 概述

本项目使用两种配置/数据管理方式：
1. **数据库字典系统** (`sys_dict_type` / `sys_dict_data`)
2. **JSON 配置文件** (`backend/app/config/json/*.json`)

两者**不冗余，互补**，服务不同的使用场景和管理需求。

---

## 对比矩阵

| 维度 | 字典系统 | JSON 配置 |
|------|---------|----------|
| **存储位置** | MySQL/SQLite 数据库 | 文件系统 (`backend/app/config/json/`) |
| **数据类型** | 简单枚举值（标签-值对） | 复杂结构化数据（嵌套对象、数组） |
| **典型示例** | 评分维度列表、角色类型、游戏模式 | 评分权重矩阵、解析策略参数、速率限制规则 |
| **管理角色** | 管理员（前端 UI 操作） | 开发者（版本控制管理） |
| **动态性** | 运行时热更新，支持启用/禁用 | 通常重启生效，开发模式支持热重载 |
| **前端消费** | 下拉选择框、标签渲染、状态映射 | 业务规则计算、API 行为控制 |
| **版本控制** | 数据由管理员控制，不纳入 Git | 纳入 Git，随代码版本发布 |

---

## 决策规则

### 使用字典系统的场景

✅ **简单枚举**：选项数量少（< 50），结构扁平  
✅ **管理员可配置**：需要非技术人员通过 UI 调整  
✅ **运行时动态性**：需要启用/禁用某项，或调整排序、颜色  
✅ **前端展示**：下拉框、标签、状态徽章等 UI 元素  

**示例**：
- `scoring_dimension` — 评分维度（伤害、治疗、增益覆盖等）
- `role_type` — 角色类型（DPS、辅助、坦克）
- `game_mode` — 游戏模式（WvW、PvE、PvP）
- `parse_status` — 解析状态标签（待解析、解析中、已完成）

### 使用 JSON 配置的场景

✅ **复杂结构化数据**：嵌套对象、多维数组、权重矩阵  
✅ **开发者控制**：需要代码审查、版本回滚、A/B 测试  
✅ **业务规则**：影响计算逻辑、算法参数  
✅ **默认值/回退**：系统启动时的兜底配置  

**示例**：
- `scoring_rules.json` — 各角色类型的评分权重（damage_weight: 0.4 等）
- `parsing_config.json` — 解析策略（API/本地/缓存 TTL/超时）
- `rate_limit_config.json` — 速率限制参数（令牌桶大小、队列长度）

---

## 反模式（避免）

❌ **把复杂权重矩阵放入字典表**：字典表不支持嵌套结构，强行扁平化会导致字段爆炸  
❌ **把简单枚举硬编码为 JSON**：失去运行时管理能力，每次修改都需要发版  
❌ **同一数据双轨存储**：避免字典表和 JSON 同时维护同一份数据

## 扩展指南

### 新增字典类型
1. 前端：在字典管理页面添加类型和项
2. 后端：如有必要，在 `get_dict_options()` 等工具函数中引用
3. 文档：在本文档的"字典系统示例"部分补充说明

### 新增 JSON 配置
1. 在 `backend/app/config/json/` 下创建 `.json` 文件
2. 在 `json_loader.py` 的 `_DEFAULTS` 中添加默认回退值
3. 在代码中通过 `load_json_config("文件名")` 读取
4. 文档：在本文档的"JSON 配置示例"部分补充说明

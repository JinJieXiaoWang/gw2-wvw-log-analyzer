激战2 WvW战场日志解析管理系统 - 完整需求文档（含AI模块）
本文档为激战2 WvW战场日志解析管理系统的完整需求说明，涵盖产品定位、核心功能（已实现+待开发）、AI智能分析模块、权限规则、设计规范、开发约束等全部内容，整合前期所有讨论要点，用于指导开发落地、原型设计、token活动申请，全程聚焦WvW大团实战场景，剔除PVE、PVP相关冗余需求。
一、产品核心定位
1.1 产品用途
本系统为激战2公会内部专属工具，核心围绕WvW战场场景，实现「日志解析-数据统计-出勤管理-技能循环-实战复盘-AI智能分析」全流程功能，服务于公会管理（出勤统计、战力评估）与团员提升（技能优化、Build适配），是轻量化、本地化、实战化的WvW数据管理与教学工具。
1.2 核心目标
- 解决公会WvW日志批量解析、团员出勤统计的效率问题，替代人工统计，降低管理成本；
- 提供技能循环教学与实战复盘功能，解决高玩循环分享不便、新手学习效率低的痛点；
- 通过AI智能分析，实现战斗短板识别、个性化优化建议，助力团员快速提升实战能力；
- 打造“数据统计-分析研判-优化落地”的闭环，提升公会整体战力与管理水平；
- 轻量化设计、本地运行，适配普通电脑配置，无需复杂部署，全员可快速上手。
1.3 技术架构
基于Python开发，后端采用FastAPI框架，前端采用Vue3 + TypeScript + Vite + Tailwind CSS + PrimeVue框架，数据库支持SQLite/MySQL/PostgreSQL三种类型（通过环境变量切换），支持.zevtc、JSON两种日志格式解析（双解析路径：dps.report API优先，失败自动降级至本地解析器），已集成AI智能分析模块（支持OpenAI/DeepSeek/Qwen多模型），整体模块化开发，确保各功能解耦、易维护、可拓展。
二、权限规则（强制执行）
2.1 登录策略
采用「非强制登录模式」，兼顾全员便捷使用与后台管理安全：
- 默认访客模式：所有公会成员无需登录，直接进入系统，无需注册、无需绑定手机/邮箱；
- 管理员登录：页面右上角设置极简登录入口，仅维护人员主动登录，账号由后台手动分配，仅支持账号密码登录；
- 权限区分：严格区分访客与管理员权限，同一页面需设计「登录前/登录后」两种状态。

三、核心功能需求（已实现+待开发）
3.1 已实现功能（开发完成，可正常使用）
3.1.1 文件上传与日志解析功能
- 支持上传解析两种格式文件：原生zevtc格式（原生战斗数据）、EI 2.59版本解析后的JSON格式；
- 两种格式解析后核心数据（技能释放、伤害、角色信息等）基本一致，确保数据准确性；
- 自动过滤非WvW无效日志，聚焦WvW场景数据提取；
- 支持单文件上传解析，解析后自动提取日志核心数据，为后续功能提供支撑。
3.1.2 字典管理功能
- 支持所有职业及对应精英特长的字典录入、编辑、删除，统一数据标准；
- 录入字段包含：职业名称、精英特长名称、专精定位，供系统其他模块调用；
- 可根据游戏版本更新灵活调整，无需修改核心代码，降低维护成本。
3.1.3 职业技能拉取功能
- 拉取所有职业及精英特长的全部技能，无遗漏；
- 拉取内容包括：技能图标、技能名称、技能描述、技能CD、技能类型（攻击/辅助/控制等）；
- 技能数据与职业、专精关联，确保精准匹配，为技能循环、复盘提供基础。
3.1.4 技能释放顺序解析与可视化测试页面
- 从解析日志中精准提取每个玩家的技能释放顺序、释放时间、技能间隔；
- 采用HTML开发测试页面，直观展示单个玩家的全部技能释放流程；
- 支持按时间顺序展示，标注技能释放时间点，可同步显示技能图标、描述，便于测试与查看。
3.1.5 系统设置功能
- 支持编辑修改系统核心信息：系统名称、系统口号、系统水印；
- 设置操作简单，编辑后实时生效，无需重启系统；
- 水印支持后续拓展显示位置、透明度，用于报表导出时的版权标识；
- 设置数据本地存储，重启系统后保留所有配置。
3.2 待开发功能（按开发优先级排序）
3.2.1 日志批量解析功能（高优先级）
- 支持选择本地文件夹，一键解析文件夹内所有zevtc、JSON格式WvW日志，自动过滤非目标格式文件；
- 解析过程显示进度提示，解析完成后给出汇总提示（成功/失败数量及原因）；
- 支持中断解析，已解析数据正常保存，未解析文件可后续继续解析；
- 批量解析数据与单文件解析标准一致，统一存入数据库，避免重复。
3.2.2 专精定制化评分模块（高优先级）
- 评分规则与职业专精精准绑定，每个专精对应独立的评分维度与权重；
- 采用外置JSON配置文件存储评分规则，支持编辑修改，无需修改核心代码；
- 自动提取日志核心数据，代入规则计算总分与分项得分，显示个人与团队平均水平的差距；
- 评分结果与玩家、战斗日志关联，可追溯每一次战斗的评分依据。
3.2.3 历史数据存储模块（高优先级）
- 完善SQLite数据库表结构，包含：玩家信息表、战斗日志表、战斗数据表、评分数据表、技能循环模板表、出勤统计表、系统设置表、AI分析记录表；
- 实现全量数据持久化，重启系统、迁移数据库文件后，所有历史数据（日志、评分、循环、出勤、AI报告）正常保留；
- 支持数据备份与恢复（可选），优化数据库查询效率，支持多条件快速查询。
3.2.4 WvW出勤统计模块（中优先级）
- 出勤判定：有有效WvW战斗日志记录（解析成功）即为出勤，无记录为缺勤；
- 支持多维度统计：按日、按周、按月统计团队整体出勤率、单个玩家出勤记录、出勤次数；
- 出勤数据与评分数据关联，可综合查看团员参与度与实战表现；
- 支持出勤记录查询、明细导出。
3.2.5 报表导出模块（中优先级）
- 支持导出三种核心报表：战斗评分报表、团员出勤报表、个人战斗详情报表；
- 导出格式为CSV（兼容Excel），字段完整，贴合WvW场景；
- 支持自定义导出路径，导出时自动添加系统水印（可选择是否显示）；
- 支持批量导出与单条导出。
3.2.6 技能循环模板管理功能（中优先级）
- 循环分类：支持理想循环（理论最优）、实战循环（适配大团场景）两种类型，均绑定职业专精；
- 循环编辑：手动新增、编辑、删除循环模板，录入技能顺序、释放时机、操作备注，关联技能数据库；
- 导入导出：支持循环模板导出为独立文件，供高玩分享；支持一键导入，自动匹配职业专精；
- 单个专精可保存多套模板，适配不同指挥打法、团战阶段。
3.2.7 技能循环可视化播放教学功能（中优先级）
- 基于HTML开发播放界面，展示技能图标、名称、释放顺序、操作备注；
- 支持播放、暂停、步进、循环重播，可调节播放速度，适配新手学习；
- 播放过程中标注技能CD、姿态切换、能量/豆子管理要求；
- 可按职业、专精、循环类型筛选模板，记录最近播放记录。
3.2.8 实战循环复盘比对功能（中优先级）
- 自动提取玩家实战技能释放记录，与选中的循环模板进行双向比对；
- 识别并标注问题：核心技能漏放、顺序错误、衔接超时、姿态/能量管理异常等；
- 生成复盘报告，包含循环执行完成度、问题明细、优化建议；
- 复盘记录存入数据库，可查询历史复盘记录，跟踪提升进度。
3.2.9 历史数据查询功能（低优先级）
- 支持多条件查询：按玩家、职业、专精、战斗日期、战斗区域、循环类型、出勤状态等；
- 查询内容涵盖：战斗数据、评分记录、出勤记录、技能释放顺序、循环复盘报告、循环模板；
- 支持分页展示、查询结果导出，可保存常用查询条件。
3.2.10 权限管理功能（低优先级）
- 角色区分：团长（管理员）、团员（访客）两种角色，支持角色分配与修改；
- 权限校验：每次操作前进行权限校验，无权限操作给出明确提示；
- 严格按2.2权限分配执行，确保核心配置与数据安全。
四、AI智能分析模块需求（核心新增模块）
4.1 AI模块核心定位
AI模块作为系统核心拓展功能，与原有功能无缝融合，聚焦WvW大团实战场景，提供智能研判、个性化优化、数据洞察服务，不替代原始数据，仅作为辅助分析工具，实现“数据统计-分析研判-优化落地”的闭环，轻量化设计，与主系统低耦合，可一键开启/隐藏。
4.2 核心AI功能模块（分模块详细说明）
4.2.1 AI战斗研判Agent
- 数据来源：依托系统日志解析后的战斗数据（伤害、治疗、buff覆盖率、技能释放、破控、削增益等）；
- 团队层面研判：自动识别团战短板（如稳固/抗性覆盖不足、破控效率低、治疗断层、削增益不及时），生成量化评估，标注问题优先级，给出团队战术调整建议；
- 个人层面研判：结合职业专精定位，对比同职业平均水平，定位个人操作短板（如秒伤偏低、关键技能漏放），输出个性化提升方案；
- 呈现形式：嵌入日志详情页，以折叠面板、图表形式展示，支持一键生成完整研判报告。
4.2.2 AI技能循环复盘Agent
- 联动模块：技能循环模板管理、日志解析（技能释放记录）；
- 核心功能：自动比对个人实战循环与理想/实战循环，智能标注失误点（漏放、顺序错误、衔接超时、姿态/能量管理异常），用不同颜色标签区分失误严重程度；
- 个性化优化：结合个人操作习惯，生成简化版实战循环模板，标注关键衔接节点、操作技巧，同步提供循环演示参考；
- 复盘报告：自动汇总循环执行完成度、失误明细、优化建议，支持导出、分享，关联跳转至循环播放页面。
4.2.3 AI Build适配优化Agent
- 联动模块：Build解析功能（装备、天赋、符文、技能解析）；
- 核心功能：针对WvW大团场景（对爆、拉扯、削拉等），解析当前Build适配性，指出短板（续航、抗性、输出/辅助失衡等）；
- 优化建议：结合公会大团作业要求，推荐天赋、装备、符文调整方案，标注调整理由；
- 多Build对比：自动研判多套Build的适配性，推荐最优方案，助力团员快速调整Build。
4.2.4 AI数据洞察Agent
- 数据来源：系统长期战斗数据、出勤数据、评分数据；
- 核心功能：

  - 战力趋势分析：分析多场战斗数据，预测团队战力波动规律，识别数据异常；
  - 出勤关联分析：研判团员出勤频次与实战表现的相关性，辅助公会人员管理；
  - 职业热度分析：统计各职业专精使用率、战力贡献，为公会职业配置提供支撑。
- 呈现形式：嵌入数据看板页、出勤统计页，以折线图、饼图+文字摘要形式展示。
4.2.5 AI报告管理与规则配置模块
- 权限限制：仅管理员可见、可操作；
- 规则配置：支持开启/关闭AI分析维度、调整评分权重，适配公会个性化需求；
- 报告管理：支持AI复盘报告批量生成、导出、删除，AI缓存清理、批量重分析；
- 模板预设：预设多套AI报告模板，可自定义话术风格，确保贴合WvW实战。
4.3 AI模块交互与视觉要求
- 视觉风格：沿用系统深色电竞风格，不异化，设定AI专属色（科技青 #00C896），用于AI按钮、标签、进度条、高亮标注；
- 交互逻辑：
        
  - AI运算过程展示实时进度+文字提示，禁止无反馈加载；
  - AI问题点位点击可自动定位到对应技能、战斗时段、图表位置；
  - AI建议条目hover显示补充说明，减少页面文字堆砌；
  - AI规则修改、报告删除等高危操作，增加二次确认弹窗。
- 空态设计：单独设计AI分析加载中、AI分析失败、暂无AI分析数据三套空态页面。
4.4 AI模块整合亮点
- 低耦合整合：AI模块独立封装，与主系统无缝联动，不影响主系统稳定，可一键开启/隐藏；
- 实战导向：所有AI输出均贴合WvW大团实战，拒绝空泛话术，可直接用于操作优化、战术调整；
- 权限适配：严格遵循系统权限规则，兼顾公开复盘学习与后台管理需求；
- 落地性强：适配Vue3+Python开发架构，无需复杂技术支撑，轻量化易维护。
五、原型设计规范（Figma高保真原型）
5.1 适配端与视觉规范
- 适配端：PC端 1920×1080，深色电竞风格，硬核游戏工具质感；
- 色彩规范：主色调 #165DFF（电竞深蓝），辅助强调色 #FF7D00（战火橙红），AI专属色 #00C896（科技青），中性色：背景 #141414、卡片 #2A2A2A、分割线 #333333、文字 #E5E5E5（常规）/#909399（次要）；

- 布局规范：12列栅格、8px基础间距、Auto Layout自动布局、统一圆角与阴影层级。
5.2 全局布局
左侧固定可折叠导航栏、顶部面包屑+右上角登录/管理员信息、右侧内容区，复杂页面（日志详情、AI复盘报告等）采用「筛选侧边栏+主内容区」双栏结构。
5.3 完整页面清单（必须全部设计）
1. 首页总览页：公会数据大盘、近期战场统计、快速功能入口（含AI复盘入口）、最新日志动态、AI团队战力分析摘要；
2. 日志管理页：zevtc文件上传区（仅管理员）、日志列表、筛选功能、解析状态标签、AI分析按钮（仅管理员）；
3. 日志详情页：原始数据概览、战斗统计、可视化图表、AI智能战斗研判模块；
4. 出勤统计页：成员列表、参战数据、多维度排序、导出功能、AI出勤关联分析；
5. 技能循环分析页：技能时序图、循环对比、失误分析、AI循环复盘模块；
6. Build解析页：代码输入框、Build详情展示、对比功能、AI Build优化建议；
7. 数据看板页：汇总数据、趋势图表、职业热度、AI数据洞察模块；
8. 管理员后台页：数据管理、解析规则配置、异常清理、AI规则配置；
9. 登录弹窗/独立登录页：极简风格，仅账号+密码+记住我+登录按钮；
10. 空状态页面：404、空数据、加载中、解析中、错误提示，以及AI专属空状态（3类）；
11. AI复盘报告页（新增）：总评、分项研判、问题明细、优化建议、导出/分享功能。
5.4 组件与交互要求
- 全局组件库：包含基础组件（按钮、输入框、表格等）与AI专属组件（AI按钮、报告卡片、优化标签等）；
- 图表组件：支持柱状图、饼图、折线图、时序图，AI图表用专属色区分；
